# 面白さ監査 R35(2026-07-21 ユーザー指示「48時間止めず、何が楽しいか一つ一つ言語化しながら調整・考案」)

方針: プレイヤーが体験する順に、各要素を「楽しいか / なぜか / どうすれば楽しくなるか」で正直に言語化。
安全な改善(ゲームUI・経済非依存)はその場で実装。経済/条件に触れるものは提案として記録し検証。
基準の芯(ユーザー確定): **選択と能動で一周の形が変わる。たまに出る大きいものが僅かずつ増えるのが面白い。**

## 要素1: クッキーのタップ / 会心
**仕様**: タップで基礎クリック力。確率で会心(倍率8+score×10)。会心率=双曲線(序盤~2%→√強い指・最高到達層で緩やかに上昇→99.5%漸近)。
段2「会心コンボ」=連続会心で倍率+3%/コンボ(30秒途切れでリセット・上限なし)。段3「余熱」=会心ごとに追加クッキー。

**楽しいか**:
- 素のタップ(数字が増えるだけ)=楽しくない。ベースライン。
- **会心=タップの面白さの源**。序盤は激レア(~2%)なので当たると「!!」の驚き(専用音・演出・"会心+N")。ユーザー既承認「たまに出る大きいやつ」。◎
- **会心コンボ(段2)=能動タップの芯になりうる**。叩き続けて会心を繋ぐと倍率が育ち、止めると切れる=「連鎖を切らすな」の緊張。◎の potential。

**楽しくない/問題**:
1. **会心コンボが完全に見えない(最大の漏れ)**: UIにコンボ数も30秒タイマーも出ない。一番能動的で面白い機構を、プレイヤーは"感じられない"。積み上がってるのに気づかない=無いのと同じ。
2. **会心率が99.5%に飽和**: 後半はほぼ毎タップ会心=「レアな驚き」が消え、会心が"ただの定数倍率"に劣化。会心を面白くしている分散(たまに大)が死ぬ。
3. **"レアで特大"の上位ティアが無い**: 素の会心が常態化すると、追いかける「稀で特大」が無くなる=興奮の天井が平ら。

**やること**:
- **[実装済✓] A. 会心コンボの見える化**(ゲームUI・経済ゼロ影響): コンボ≥2でクッキー上部に「🔥コンボ ×N」+30秒ゲージ(会心ごとに弾む・尽きると消える)。
  能動タップの手応えを"見える"化=芯を起こす。Playwright確認: 5連コンボ表示・失効で非表示・JSエラー無し。branchに実装(段2「会心コンボ」取得時のみ表示)。
- **[実装済✓] B. 超会心(レア特大会心)**: 会心の3%が特大(×4)、残り97%を僅かに小さく(×0.907)=**平均倍率不変の再分配**。
  simは期待値計算なので分散再分配は不可視=**経済/バランス影響ゼロ(平均保存=1.000000で検証)**。特大フロート+金フラッシュ演出。
  後半で素の会心が常態化しても「たまに出る特大」を永久に残す。ユーザー最愛の形をリスクゼロで実装。E2E合格・JSエラー無し。
  **手法の発見**: "稀で特大"は平均保存の再分配で作れば経済に一切触らない=横断展開(金/ドロップ)にも同じ手が使える。
- **[検討] C. 会心率の飽和を99.5%まで上げない**: 会心を"時々"に保ち、常態化はコンボ/超会心で作る。ユーザーが上限で二転三転した経緯あり=勝手に触らず要相談。

### メタ発見(全要素共通): 「たまに出る大きいやつ」は横断パターンにできる
ユーザー最愛の形=稀で特大の驚き。会心(超会心)・金クッキー(大当たり)・素材/装備(レアドロップ)・ボス(特大報酬)に
**同じ"稀で特大"の演出言語を散らす**と、ゲーム全体が「いつ何が跳ねるか分からない」期待で満ちる。要素を一巡してから一貫実装する。

## 要素2: 設備購入 / 進行ループ
**やること**: クッキーを貯めて次の設備/研究を買う→cps上昇→また貯める。
**楽しいか**:
- 放置ゲーの核の快感=「やっと買える」瞬間とcpsが跳ねる手応え。これ自体は◯(在る)。
- **不満の核(ユーザー「個数変えるやつだけ買うだけ」)= 選択が無い**。クッキーは結局全部買えるので**機会費用がない**=AかBかの本物の悩みが生まれない。全部「+N」で質的な違いを感じにくい。
**問題の分類**:
1. 機会費用ゼロ(構造)= 選択深度の欠如。→ これは「選択と能動で形が変わる」の本丸(R34提案: ㉘'/②改緩和で型の峰を立てる)。購入単体でなく型・研究機構の意味付けで解く。
2. 進行の"手応え(juice)"が弱い= 新機構/新モンスター/新設備を解放した瞬間の祝福演出が薄い。解放ドリップ(㉚)で新着は演出できる素地あり。→ 安全なjuice追加候補(後述で横断実装)。
**やること**: 購入単体に新機構は足さない(機会費用の欠如は型/条件側=R34提案で解く)。進行の節目のjuiceは横断juiceで拾う。

## 要素3: 金クッキー
**やること**: 金クッキーがランダム位置に出現→タップで捕まえる→50%即時獲得/50%全生産ブースト。spice熟成・echo「金は金を呼ぶ」(35%で追い金)。
**楽しいか**:
- **ランダム位置の"捕まえる"=◯**(能動の小さな喜び・classic clicker)。echo連鎖「金は金を呼ぶ!」も小さなストリークで◯。
**問題**:
- **大当たり(jackpot)が無い**: 毎回ほぼ同じ大きさ(cps基準)。稀で特大の"!!"が無い=ユーザー最愛の形の欠落。金は jackpot の恰好の置き場。
- 50/50即時/ブーストはランダムで選択でない(捕まえる行為としてはOK)。
**やること**: **[実装済✓] 金の大当たり**=即時金の3%が×5、残り×0.876で平均保存=経済ゼロ影響(平均=1.000000検証)。稀に「💰金 大当たり」+フラッシュ。超会心と同じ平均保存の手法。

## 要素4: 戦闘(モンスター討伐)
**やること**: モンスター出現→タップで撃破(最小タップ床で複数回・HPバー)→**討伐後に報酬カード3枚以上から1枚選ぶ**。型相性(戦車→装備素材等)・オーバーキル(残HP×5)→レア素材・討伐記録の節目祝福・ボス(25討伐ごと・報酬選択+1)。
**楽しいか**:
- **報酬カード選択(3枚→1枚)=ゲーム最大の能動決定**。毎討伐で周回の方向を選べる=◎(agencyの本丸)。
- 討伐記録の節目祝福・金獣変異(ブースト中に稀に報酬Lv+1)・オーバーキル="稀で特大"の種は在る。◯
- 最小タップ床でワンパン虚無は解消(土台)。
**問題**:
1. **タップそのものは反復**(同じ敵を4回)=面白さは討伐の"周り"(選択・ドロップ)に在り、タップ内には無い。
2. **型相性が見えているか怪しい**: 「戦車を狩ると装備素材」をプレイヤーが知らなければ、狩る相手を選ぶ決定が不可視。→要確認・要見える化。
3. **報酬カードが似通うと選択が形骸化**: 3枚が全部似た+N perkだと"選んでる感"が薄い。→カードの質的差と、このカードが今周回に何をするかの可読性。
4. **ボスが"イベント"になっていない可能性**: 16タップ+報酬+1はあるが、特別感の演出が薄いかも。→ボスを大イベント化。
**やること(候補)**:
- [要確認→見える化] 型相性(どの敵が何に効くか)をプレイ画面で示す=狩る相手を選ぶ決定を起こす。
- [提案・横断] オーバーキル/ボスに"稀で特大"の報酬と演出。ボスを節目の大イベントに。

## 監査の全体像(要素1-4で見えた構造)
骨は良い(会心・金・討伐選択・ボス・型)。**面白さの漏れは3種**:
(1)**良い機構が不可視**(会心コンボ=修正済✓ / 型相性 / …)。
(2)**"稀で特大"が弱い**(超会心・金大当たり・レアドロップ・ボス=横断で恒久化)。
(3)**節目のjuiceが薄い**(解放・討伐記録・ボス撃破の祝福演出)。
→ これらは大半がゲーム側(経済非依存)で安全に足せる。"稀で特大"だけ経済に触るのでsim検証。
**手法の確立(実装で判明)**: "稀で特大"は**平均保存の再分配**で作れば経済ゼロ影響。会心・金で実装済。ドロップにも展開可。

## 要素5: ボス
**やること**: 討伐N体ごと(周期25+10×層)にボス出現。HP×12(最小タップ床で16タップ=長めの戦い)。ボス核ドロップ・報酬選択+1。
**楽しいか**: 周期的な大物=イベントの素地は在る。最小タップ床で歯応えも出た。
**問題**:
1. **予兆が不可視**: 「ボス出現まであと3体」は統計パネルの中だけ=プレイ画面に無い→ボスが不意打ちで、盛り上がり(あと少し!)が消えている。会心コンボと同じ不可視の漏れ。
2. **撃破の特別感が薄いかも**: ボス核+報酬+1はあるが「ボス撃破!」の祝福・稀で特大が弱い。
**やること**: **[実装済✓] ボス接近インジケータ**=プレイ画面に「👑 ボスまであと N体」(N≤5で出現・N≤2で脈動)+ボス在場中は「👑 ボス出現!」。不意打ちを"期待"に変える。純UI・経済ゼロ影響・E2E合格。

**やること**: [提案] レア素材ドロップに"稀で特大"の演出(横断)。装備の効果差の可読性は要確認。

## 要素7: 研究(段階式 / 実績研究 / スキル解放研究)
**楽しいか**:
- **段階式研究(段2/段3)=新機構が付く=質的変化で◎**(会心コンボ・延長狩り・利息・余熱など)。買う価値のある"面白い購入"。
- **実績研究(数百枚の×1.3系)=ノイズ**。「個数買うだけ」の主犯格の一つ。達成で解放され、常に即買い=決定ですらない。
**問題**: 実績研究の物量が"無心の連打買い"を生む。解放ドリップ(㉚)でペースは付いたが、質は変わらない。
**やること**:
- [提案] 実績研究は**自動購入**(常時即買い=決定でないなら、手動連打から解放してノイズを消す)。プレイヤーの手は段階式・スキル・報酬など"選ぶ"購入に集中。=「買うだけ」の体感を大きく削る。ゲーム挙動変更=要ユーザー確認。
- [維持] 段階式研究は面白いので厚く。1研究1増幅変数の原則(既存)。

## 要素8: スキル(転生時のツリー)
**楽しいか**: ツリーでノード選択=build構築の決定=◎の素地。方針署名スキルで型の色。
**検証済**: スキルは**転生ポイントを予算にノードを買う**(prestige -= skill.cost)+プレレク(木)+署名スキル排他=**予算制約の本物のbuild選択**(「全部タダ」でない)。選択システムとして健全◎。
**残る問い**: 可読性(何が今買う価値かが見えるか)。ツリーが広く、初見で"どこを目指すか"が立てにくいと選択が迷子になる。
**やること**: [要確認] ツリーUIの可読性(推奨ルート/署名の明示)。構造は良いので触るなら見せ方。

## 要素9: 転生 + プレイ方針(型選択)
**楽しいか**: **「計画を持って仕切り直す」瞬間+型を選ぶ=ゲーム最大の戦略決定=◎の本丸**。ユーザーの芯「選択で形が変わる」の中心。
**問題(R34の核心)**: 型は収入内訳では実際に違う形になっている(hunt=討伐/golden=金…)が、**プレイヤーがその違いを"感じる"設計になっているか**。②改/㉘の均一化が型の峰を平らにして隠していた(R34提案で外す)。
**やること**: R34提案(㉘'山だけ/②改緩和/balanced専用)=型の峰を立てて"選んでる感"を出す。**要ユーザー判断**。加えて[考案] 型ごとの体験差を"見せる"(この型はこれが主役、の明示)。

## 要素10: タイミング機構(圧縮チャージ / 熟成 / 観測ゆらぎ / 延長狩り)
**楽しいか**: **能動タイミング=ユーザーの芯"能動"そのもの**。チャージを溜めて放つ・波の山に合わせる=◎の素地。圧縮チャージはbhGaugeBtn(クッキー横)で可視・タップ発動=良い。
**問題**:
1. **可視性のムラ**: 圧縮チャージは見える(◎)が、熟成(前回金からの経過)・観測ゆらぎ(90秒周期の山)は不可視=能動の判断ができない。会心コンボと同じ漏れ。
2. **後半ゲート**: 全部段2研究=中盤以降しか無い。序盤に能動タイミングの芯が無い。
3. **延長狩り**: 測定snapが末期で1.000(R34)。実プレイでは働くが手応えが弱い可能性。
**やること**: [実装候補] 熟成/観測ゆらぎの状態を見える化(次の山まで/熟成ゲージ)=能動タイミングの判断を起こす。純UI。

## 監査サマリ(要素1-12・第一巡)
**実装済み(全て経済ゼロ影響・E2E合格・branch)**: ①会心コンボ可視化 / ①B超会心 / ③金の大当たり / ⑤ボス接近インジケータ。

**次の安全な実装候補**: 熟成/観測ゆらぎの可視化・レア素材ドロップ演出・(要確認後)スキルツリー/装備の可読性・型の主役の明示。
