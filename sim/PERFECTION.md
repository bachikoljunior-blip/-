# 完璧の定義(=止まってよい唯一の条件)

このバーが**全部 pass するまでループは止まらない**。1つでも未達なら BACKLOG から次を引いて作業を続ける。
「もう十分」「Go待ち」「方向をくれ」で止まるのは再発(ANTI_REGRESSION#1/#2)。止まる=全項 pass のときだけ。

## バー(各項 checkable・状態を更新する)
- **P1 全事象OK**: 実プレイで各事象が発火し、演出/数値/feltが北極星「選択と能動で一周の形が変わる」に適う。
  対象: タップ/会心/会心コンボ/★フィーバー/超会心/金/金チェイン/★ゴールドラッシュ/討伐/オーバーキル/ボス/報酬カード/実績/研究解放/設備購入/層上げ/装備作成/転生/スキル取得/注文/料理。
  検証: イベント実況(時間+画面付き)で一つ一つ確認。 … 状態: 進行中(料理/注文/オーバーキル未確認)
- **P2 各画面OK**: タイトル/最序盤/序盤/中盤/深層/エンドの各画面に、破綻・混乱表示・見切れ・空表示・潰れた楽しさが無い。
  検証: 実画面スクショQA。 … 状態: 主要は確認済(grace/ヒント修正済)。全画面一巡は未完
- **P3 経済ニュートラル**: sim バッテリー(income/expect/baseline)が緑=経済に触れてない/触れたら平均保存で不可視。 … 状態: fever/rush で確認済。以後の変更ごとに再確認
- **P4 既知の軟点ゼロ**: 記録した軟点が全て解消。 … 状態: **pass**(「最序盤の単調」はMAX買い実ログで否定=事象豊富・×1買いdriverのartifactだった。新規軟点が出たら再open)
- **P5 北極星奉仕**: 全変更が「選択と能動で一周の形が変わる」に奉仕。反する/無意味な追加ゼロ。 … 状態: 維持

## 判定
P1〜P5 全て pass、かつ直近のQA一巡で新規issueゼロ → **完璧**。トリガー停止・ユーザーへ報告。
それ以外 → 止まらない。BACKLOG の先頭を引いて作業。
