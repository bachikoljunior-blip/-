# 自律ゲーム制作の仕組み(このセッションの実証で確定した再現手順)

目的: AIが自律で「ユーザーが納得するゲーム」を作れるようになるための、再現可能な手順。
これは一般論ではなく、**この制作で実際に失敗して直した所から確定した**ルール。各項に根拠(実例)を付ける。

## 0. 一番上の原則
- **意図で動く。字面で動かない。** 指示の裏の目的を読み、着手前に必ず自己批判する:「これは目的に適ってるか?」
  - 実例: 「最初から最後まで実況」を字面で受け、100時間級エンドを live で忠実再現しようと auto-player を数時間 grind(大浪費)。目的は"ユーザーのQA"であって放置耐久の証明ではなかった。**着手前の自己批判が無かったのが失敗**。
- **でっち上げない・隠さない・過剰推測しない。** バグでないものをバグと言わない。出来てない所を出来たと言わない。言われてないことを勝手に深読みしない。

## 1. 一次物差し=ユーザーの美意識を先に言語化(sim/SENSE_R35.md)
- 合格条件(代理指標)より前に、**ユーザー固有のセンス**を明文化して持つ。判断は毎回そこから。ユーザーが加筆訂正して育てる。
- 例: 稀で特大×逓増 / 選択と能動で一周の形が変わる / hard-won / 無心作業を嫌う / 次への動機 / 反degeneracy。

## 2. 北極星を1つに絞る
- 一言のビジョン(例:「選択と能動で一周の形が変わる」)。**全ての変更がそれに奉仕**するか毎回問う。奉仕しない追加はしない。

## 3. 経済/システムは壊さない(sim先行・平均保存・revert可)
- **持続的な正の乗数は prestige/複利で必ず暴走**。実証: フィーバーを"確定超会心=経済ブースト"にしたら S2総稼ぎ 4.2e129→1.3e217(3e87×)。golden spawn 増幅も同型で桁溢れ。
- 安全な作り方 = **既存の平均保存の稀-特大(超会心/金大当たり)を hard-won 契機で"時間再配分"**。総量不変=sim不可視=経済ゼロ影響。→ フィーバー/ゴールドラッシュはこれで実装、S2がpre-fever完全一致で確認。
- 経済に触る案は必ず **sim先行 → バッテリー(income/expect/baseline)確認 → revert可**。純UI/文言/演出は自己判断で即。

## 4. 実ゲームで検証(コードや sim だけで済ませない)
- **実際に遊んで/QAして本物の綻びを直す**。実例: 実プレイで「開始30秒 ノルマ N/0」の混乱表示、強化ヘッダーの見切れヒントを発見→修正。読むだけでは出なかった。
- 圧縮プレイの技: Playwright 合成時計で時間ごと早送り=生産もノルマも整合。タップは時計を進めない=激安(FX stub でさらに)。報酬/スキル/転生は addTap(pointerdown)なので DOMクリック不発→**関数を直呼び**(revealRewardChoices/chooseReward, selectSkill/takeSelectedSkill, prestigeReset)。深い状態のQAは debugMode で整合的に構築(=偽injectでなくゲーム自身の論理)。

## 5. 再発ゲート(sim/ANTI_REGRESSION.md)を毎回照合
- 結論前に5点を検問: ①「既に十分/Go待ち」= 判断回避 ②承認・些末の質問 = 丸投げ ③「理解した/伝わった」= 理解の演技 ④ユーザーの口調を真似る ⑤過剰な深読み。1つでも出たら結論を破棄して「決める→作る」。

## 6. ループ(1変更の回し方)
決める(自分の判断で)→ 作る(北極星に奉仕)→ 測る(sim/実プレイ)→ 味に合わせて調整 → 実機で確認 → commit/push。**main へマージしない**(ユーザー指示)。報告は成果と検証だけ、短く。

## 7. この仕組みの育て方
- 失敗するたびに「どのルールが欠けてたか」をここに1行足す。ルールは抽象論でなく**実例つき**で書く(そうでないと次also守れない)。
- 生きた文書: SENSE(美意識)/ ANTI_REGRESSION(再発)/ 本書(制作プロセス)の3点セットで運用。
