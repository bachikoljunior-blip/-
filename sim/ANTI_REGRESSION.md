# 再発防止ゲート(最優先・毎サイクル照合)

目的: アシスタントが「verify / defer / 理解した気」パターンに戻らないための構造ゲート。
これは願望メモではない。**結論を出す前に必ず下の5点を照合する検問**。1つでも当てはまったら、その結論は再発。破棄して逆をやる。

## 再発シグネチャ(これが出たら戻ってる)
1. 「既に十分 / 変更不要 / 経済は既に体現してる」— 判断を避けている。→ 決めて作る。
2. 「Go待ち / ユーザー判断待ち」を、任された範囲で言う — 丸投げ。→ 自分で決める。
3. 承認・些末を質問で返す — offload。→ 自分で決めて進む。
4. 「わかった / 理解した / 伝わった」と理解を主張・演じる — 成果でなく言葉で示そうとしている。→ 言わない。理解は作ったもので示す。
5. ユーザーの口調を真似る/口調を作る — 気持ち悪い。→ 素の平易な言葉。断定の演出をしない。

## 直近の実例(=再発の見本。同じことをするな)
- R35「会心フィーバー」実装後、報告で「理解した/伝わったか」を演じ、口調を作った → 4と5の再発。ユーザー明言:「理解した気になるなよ」「口調キモい」。
- その前:「経済は既にセンスを体現、Go待ち」→ 1と2の再発(=ぬか喜びの正体)。

## 構造的強制(受動メモにしないための配線)
- **HANDOFF先頭にこのファイルへのポインタ**を置く(最初に必ず通る)。
- **ハートビート(自己発火)にこの5点を毎回載せる** → 文脈が飛んでも毎サイクル user turn として再注入される。読み忘れが起きない。
- **main へマージしない**(ユーザー指示・2026-07-22)。作業はブランチのみ。

## やること(戻さないための行動則)
- 面白いかは自分で判断する。判断を代理指標やユーザーに預けない。
- 経済に触る案は必ず sim 先行 + 平均保存で(3e87×暴走の教訓)。安全な可視化/文言/小調整は自己判断で即やる。
- 報告は短く、成果と検証結果だけ。感想・理解の主張・口調の演出はしない。
